Context
بناء منصة دعم عملاء مدعومة بالذكاء الاصطناعي مشابهة لـ mojeeb.app، موجهة للشرق الأوسط. المنصة تتضمن دردشة ذكية متعددة القنوات (WhatsApp, Messenger, Instagram)، كشف المشاعر، توجيه ذكي، استخراج عملاء محتملين، تحليلات، واشتراكات مدفوعة.

Monorepo: Turborepo + pnpm
Real-time: Socket.IO + Redis adapter
Containerization: Docker Compose

Backend Architecture
API Routes (/api/v1/)
Auth: register, login, refresh, logout, forgot/reset password, verify email
Organizations: CRUD + invite/manage members
Agents: CRUD + test endpoint
Channels: CRUD + agent assignment
Conversations: list (filterable), detail, handoff, resolve
Messages: list (paginated), send as human agent
Knowledge Base: CRUD + document upload/manage
Leads: list, update status, export CSV
Analytics: overview, conversations, emotions, response time, agents
Subscriptions: checkout, change plan, cancel, invoices, usage
API Keys: create, list, revoke
Webhooks: WhatsApp, Messenger, Instagram, Kashier (signature-verified)
Auth Flow
JWT (RS256) access tokens (15min) + refresh tokens (7d, hashed in DB)
Backend issues tokens, frontend uses NextAuth v5 Credentials Provider
Org context middleware injects membership/role from JWT
Queue System (BullMQ + Redis)
inbound-messages: Parse channel payload → store message → trigger AI
ai-processing: Generate response, detect emotion, extract leads, routing decision
outbound-messages: Send via channel adapter with retry (3 attempts, exponential backoff)
analytics: Async event processing and counter updates
AI Architecture
Provider Pattern: Abstract AIProvider base class with OpenAIProvider and AnthropicProvider
Factory: getAIProvider(agent.aiProvider) returns correct implementation
Pipelines:
ResponsePipeline: Context building → KB retrieval (pgvector) → AI generation → parallel analysis
EmotionPipeline: Detect emotion from message + context (JSON structured output)
RoutingPipeline: Rule-based checks first, then AI decision for ambiguous cases
LeadPipeline: Extract contact info, interests, intent from conversation
Channel Adapter Pattern
Abstract ChannelAdapter with: verifyWebhook(), parseInbound(), sendMessage(), markAsRead()
Implementations: WhatsAppAdapter, MessengerAdapter, InstagramAdapter, WebchatAdapter
All Meta channels share webhook verification pattern (HMAC SHA-256)
WebSocket (Socket.IO)
/dashboard namespace: conversation events, notifications
/chat namespace: live messages, typing indicators
Redis adapter for horizontal scaling
Frontend Architecture
i18n + RTL
